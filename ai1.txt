## 🛑 Category A: Core Physics & Propagation (grid.py)

**1. The "Bidirectional Failure" Bug**
* **Issue:** Currently, if node `P1` fails, `P0` (upstream) might appear isolated because your graph search is undirected or lacks strict flow hierarchy.
* **Implementation:** Force `self.graph = nx.DiGraph()`. Ensure `get_downstream_nodes` only looks at `successors`.

**2. Flow-Source Inconsistency**
* **Issue:** Edges don't know *what* energy is inside them. They only know the quantity (`flow`).
* **Implementation:** During the `forward_sweep`, carry the `source_type` (solar, nuclear, etc.) from the parent node to the child edge.

**3. Resistance-Voltage Disconnect**
* **Issue:** Your voltage calculation is a simplified drift. Real voltage drops are proportional to $I \times R$ (Current $\times$ Resistance).
* **Implementation:** Update `_update_stress` to calculate $V_{drop} = \sum (Flow_{edge} \times Resistance_{edge})$.

**4. Reactive Power ($\text{VAr}$) Absence**
* **Issue:** You only model Active Power (MW). Real grids fail due to Reactive Power (Voltage) collapse.
* **Implementation:** Add a `reactive_load` attribute to nodes (usually 20% of `load`).

**5. Inertia Modeling**
* **Issue:** Frequency drops to 0 instantly when a generator fails. Real generators have spinning mass (inertia).
* **Implementation:** Use a decay constant: `freq = freq_old - (imbalance / inertia)`.

**6. Static Line Limits**
* **Issue:** Many edges have a hardcoded `capacity=1.0`. Real wires vary significantly by gauge.
* **Implementation:** Assign capacities based on hierarchy: Substation-to-Transformer (50MW), Pole-to-House (0.5MW).

## 🔋 Category B: EMS & Energy Logic (ems.py)

**7. Battery "Generator" Ghosting**
* **Issue:** Batteries are treated as loads that "disappear" when full. They should behave as voltage-supporting sources.
* **Implementation:** When `ems` discharges a battery, set `node.generation += discharge_amount` so the Physics engine sees it.

**8. Lack of "Baseload" vs. "Peaker" logic**
* **Issue:** Nuclear and Coal ramp at the same speed. Nuclear should be slow; Gas/Battery should be instant.
* **Implementation:** Add a `ramp_rate` variable to `GridNode`.

**9. Economic Dispatch Gap**
* **Issue:** EMS uses whatever is available. It should use the cheapest (Renewables) first, then Nuclear, then Coal.
* **Implementation:** Sort active generators by `marginal_cost` before allocating load.

**10. Supercapacitor Over-utilization**
* **Issue:** Supercaps are drained for sustained loads. They should only trigger for $\Delta V$ (sudden voltage spikes/dips).
* **Implementation:** Gate supercap discharge: `if abs(v_prev - v_curr) > 0.05: use_supercap()`.

**11. Grid-Import Blindness**
* **Issue:** If the system is in deficit, it just shows a negative balance. It should simulate "Buying from the Main Grid."
* **Implementation:** Create a virtual node `EXTERNAL_GRID` that provides infinite power but costs $500\text{/MW}$.

## 🤖 Category C: SCADA & AI Control (scada.py)

**12. The "Magic" Reroute**
* **Issue:** Rerouting happens between any two nodes. In reality, it can **only** happen if a `tie-switch` exists.
* **Implementation:** In `_flisr_restore`, filter `all_simple_paths` to only include paths containing an edge where `is_tie_switch == True`.

**13. Non-Sequential Switching**
* **Issue:** All switches move at once. Real SCADA closes switches in sequence to check for "fault re-entry."
* **Implementation:** Use a `time.sleep` or timestep delay between `ISOLATE` and `RESTORE` actions.

**14. LSTM Look-ahead Ignoring Weather**
* **Issue:** The LSTM predicts load based on history but doesn't weigh the "Storm" flag heavily enough.
* **Implementation:** Add `weather` as a primary feature column in the LSTM input tensor.

**15. DQN Action Space Overlap**
* **Issue:** Action 0 (Generation) and Action 1 (Battery) do the same thing: increase supply. The AI gets confused.
* **Implementation:** Penalize Battery use in the reward function if `marginal_cost` of Coal is currently lower.

**16. "Soft" Isolation**
* **Issue:** You mark nodes as `isolated`, but you don't physically "Open" the breakers in the graph data.
* **Implementation:** When a fault is isolated, set `edge["active"] = False` for the boundary switches.

## 🎨 Category D: Frontend & Visualization (GridGraph.jsx)

**17. Particle-Based Flow (The "Dot" Problem)**
* **Issue:** You use moving dots. It looks like a toy.
* **Implementation:** Use **Stroke-Dash-Offset** animation on the lines themselves to show "Liquid Electricity."

**18. Color Disconnect**
* **Issue:** Wires are one color. They should be colored by the **Source Type** (Solar = Yellow, Battery = Purple).
* **Implementation:** Read `edge.source_type` in the D3 render loop.

**19. Voltage Heatmap Missing**
* **Issue:** You can't see "brownouts" (low voltage).
* **Implementation:** Change node stroke colors based on voltage: $1.0 = \text{Green}$, $0.9 = \text{Orange}$, $0.8 = \text{Red}$.

**20. No "Switch Status" indicators**
* **Issue:** You can't see if a breaker is Open or Closed visually.
* **Implementation:** Draw a "Gap" or a "Cross" on edges where `active == False`.

**21. Ghost Flow**
* **Issue:** Sometimes flow animates on dead lines.
* **Implementation:** `if (!edge.active) return;` inside the canvas animation function.

```python
def _flisr_restore(self, grid):
    # Only find paths that close an ACTUAL tie switch
    tie_edges = [(u, v) for u, v, d in grid.graph.edges(data=True) 
                 if d.get("is_tie_switch") and not d["active"]]
    
    for u, v in tie_edges:
        if self.can_connect_islands(u, v):
            grid.graph[u][v]["active"] = True
            grid.graph[u][v]["switch_status"] = "closed"
            return f"FLISR: Closed Tie-Switch {u}-{v}"
```
